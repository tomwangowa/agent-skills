Welcome! This is a 10-minute introduction to Docker for developers. We'll cover the fundamentals and get you ready to start using containers.

What is Docker?

Docker is an open-source platform that automates the deployment of applications inside lightweight, portable containers.

  • Containers: Isolated environments that package code and dependencies
  • Portable: Run consistently across different environments
  • Lightweight: Share OS kernel, start in seconds
  • Scalable: Easy to replicate and orchestrate
Docker revolutionized application deployment by providing a standardized way to package and run applications. Unlike VMs, containers share the host OS kernel, making them much lighter and faster.

Why Use Docker?

PROBLEM

  • "Works on my machine"
  • Environment inconsistencies
  • Complex dependencies
  • Slow deployment process

SOLUTION

  • ✓ Consistent environments
  • ✓ Fast, reliable deployments
  • ✓ Simplified dependencies
  • ✓ Infrastructure as code

KEY BENEFIT

"Build once, run anywhere" - from local dev to production

Docker solves the classic "works on my machine" problem by ensuring that your application runs the same way in development, testing, and production environments.

Core Concepts

IMAGE

A read-only template with instructions for creating a container. Built from a Dockerfile.

CONTAINER

A runnable instance of an image. Isolated process with its own filesystem, networking, and resources.

DOCKERFILE

A text file containing commands to build a Docker image. Defines the application environment.

REGISTRY

A repository for storing and distributing Docker images (e.g., Docker Hub, private registries).

These four concepts are fundamental to understanding Docker. Images are like classes, containers are like instances of those classes.

Essential Commands


# Build an image from Dockerfile
docker build -t my-app:1.0 .
docker images

# Run a container
docker run -d -p 8080:80 --name my-app my-app:1.0
docker ps

# Stop and remove container
docker stop my-app
docker rm my-app

# Pull/push images
docker pull nginx:latest
docker push myregistry/my-app:1.0

COMMON OPTIONS

-d detached mode

-p port mapping

USEFUL COMMANDS

docker logs view logs

docker exec run commands

These are the commands you'll use most frequently. Build creates images, run starts containers, and the registry commands let you share images with your team.

Example: Node.js App


# Start from Node.js base image
FROM node:18-alpine

# Set working directory
WORKDIR /app

# Copy package files
COPY package*.json ./

# Install dependencies
RUN npm ci --only=production

# Copy application code
COPY . .

# Expose port and start app
EXPOSE 3000
CMD ["node", "server.js"]

BUILD & RUN

docker build -t my-node-app .
docker run -p 3000:3000 my-node-app
This Dockerfile shows best practices: use official base images, copy dependencies first for better caching, use multi-stage builds in production, and specify exact commands to run.
